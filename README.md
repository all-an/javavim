Requirements

    JDK (Java Development Kit): Required for compiling and running Java code.
    GCC (GNU Compiler Collection): Required for compiling C++ code.
    JUnit and Hamcrest: Included in the lib/ directory for Java testing.

Notes

    The project uses custom test framework for C++, and JUnit for Java.
    The project is structured to support easy integration of additional features like a terminal-based interface, keybindings, and more.

Project Goals:

    Implementing a terminal-based interface for the code editor.
    Adding Vim-like keybindings.
    Integrating Git functionality similar to Visual Studio Code's built-in features.
    Expanding testing frameworks for both Java and C++.

Project extra Goals:

    Learn JNI (Java Native Interface).
    Improve Java knowledge.
    Learn C++.
    Learn how to create a Java code editor in the terminal using Eclipse JDT.

Project Structure:

    Java and C++ Integration: Utilizing JNI for native method calls.
    Custom Test Frameworks: Implementing simple, custom test frameworks for both Java and C++.
    Minimalistic and Terminal-Based Approach: The project focuses on building a minimalistic code editor that runs in the terminal.
    Vim-like Keybindings: The project will include Vim-like keybindings by default.
    Integrated Terminal: The editor will have an integrated terminal within it.
    Git Integration: The project will include Git integration similar to Visual Studio Code's built-in Git features.
    File Explorer: Similar to Vim's NerdTree, the project will include a file explorer feature for easy navigation and file management within the editor.
    Build Tools: The project will avoid using Maven, Gradle, or similar build tools; all build processes will be managed manually or with custom scripts.
    Testing: JUnit for testing Java code and a custom test framework for testing C++ code.
    Debugging: GDB for debugging C++ parts and JDB for debugging Java parts.
    Java Libraries:
        Picocli: For command-line interface (CLI) functionality.
        JLine or Lanterna: For terminal-based UI management.

Setup and Usage

1. Compile and Run the Project

To compile and run the project, use the run.sh script:

```
bash run.sh

```

2. Run the Java Tests

To compile and run the Java tests, use the test_java.sh script:

```
bash test_java.sh

```

3. Run the C++ Tests

To compile and run the C++ tests, use the test_cpp.sh script:

```
bash test_cpp.sh

```

## Javavim first project structure

## Overview

Javavim is a minimalistic terminal-based code editor project implemented using Java and C++ with JNI integration. It serves as a learning project to:

- Learn JNI (Java Native Interface)
- Improve Java knowledge
- Learn C++
- Learn how to create a Java code editor in the terminal using Eclipse JDT

## Project Structure

```plaintext
Javavim/
│
├── src/
│   ├── java/
│   │   ├── Javavim.java             # Main Java class with JNI integration
│   │   └── Javavim.h                # JNI header file (generated by javac)
│   └── cpp/
│       ├── Javavim.cpp              # C++ file implementing JNI method
│       ├── MessagePrinter.cpp     # C++ file with a pure C++ function
│       └── MessagePrinter.h       # Header file for the pure C++ function
│
├── test/
│   ├── javatest/
│   │   └── JavavimTest.java         # Java test class using JUnit
│   └── cpptest/
│       ├── JavavimTest.cpp          # C++ test file using a custom test framework
│       └── JavavimTest.h            # Header for the custom C++ test framework
│
├── lib/
│   ├── junit-4.13.2.jar           # JUnit library for testing
│   ├── hamcrest-core-1.3.jar      # Hamcrest library required by JUnit
│   ├── jline-3.21.0.jar           # JLine library (example)
│   ├── lanterna-3.1.1.jar         # Lanterna library (example)
│   ├── picocli-4.6.3.jar          # Picocli library (example)
│
├── build/                         # Directory where compiled files will go
│
├── run.sh                         # Script to compile and run the project
├── test_java.sh                   # Script to run the Java tests
├── test_cpp.sh                    # Script to run the C++ tests
└── README.md                      # Project documentation
```
